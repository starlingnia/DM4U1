# U(1) Lattice Gauge Theory - Physics-Conditioned Diffusion Models: Testing & Running Guide

This guide describes the project structure, how to set up the environment, how to run the automated unit tests, and how to execute the notebook simulations.

---

## 1. Project Overview & File Structure

This project applies time-dependent score-based Diffusion Models to $U(1)$ Lattice Gauge Field Theory. It includes implementations for training a model on lattice configurations generated by Hybrid Monte Carlo (HMC) and sampling new configurations using Metropolis-Adjusted Langevin Algorithm (MALA) or ODE solvers.

Here is the role of each module:

*   **[unet.py](file:///Users/junxiongnie/code/DM4U1/unet.py)**: Defines `ScoreNet`, a time-dependent U-Net neural network architecture that models the score function $\nabla_x \log p_t(x)$ conditioned on the noise level $t$.
*   **[sde.py](file:///Users/junxiongnie/code/DM4U1/sde.py)**: Defines SDE properties such as the marginal probability standard deviation (`marginal_prob_std`) and diffusion coefficient (`diffusion_coeff`) for variance exploding/preserving perturbations.
*   **[loss.py](file:///Users/junxiongnie/code/DM4U1/loss.py)**: Calculates the denoising score matching loss (`loss_fn`) used to train the score-based generative model.
*   **[mc.py](file:///Users/junxiongnie/code/DM4U1/mc.py)**: Contains pure Markov Chain Monte Carlo (MCMC) functions, including U(1) gauge action (`get_action`), drift force (`get_drift`), gauge cooling (`gauge_cooling`), and Hybrid Monte Carlo (`hmc`) update steps.
*   **[sampler.py](file:///Users/junxiongnie/code/DM4U1/sampler.py)**: Contains the sampling routines:
    *   `ode_sampler`: Samples configurations via black-box ODE solvers.
    *   `MALA_sampler`: Samples configurations using Metropolis-Adjusted Langevin Algorithm guided by the trained score model.
    *   `calculate_topological_charge`: Calculates topological charge on a 2D U(1) lattice.
*   **[observable.py](file:///Users/junxiongnie/code/DM4U1/observable.py)**: Provides statistical measurement tools:
    *   `jackknife_stats`: Jackknife error analysis.
    *   `compute_autocorrelation` / `compute_ess`: Calculates integrated autocorrelation time and Effective Sample Size (ESS) to quantify MCMC sampling efficiency.
*   **[norm.py](file:///Users/junxiongnie/code/DM4U1/norm.py)**: Maps lattice gauge fields (angles) from $[-\pi, \pi]$ to $[-1, 1]$ for neural network normalization.
*   **[test_project.py](file:///Users/junxiongnie/code/DM4U1/test_project.py)**: The automated unit test suite verifying each module's correctness.

## 2. Environment Setup

The code requires **Python >= 3.8** and **PyTorch >= 2.0**.

> [!IMPORTANT]
> **NumPy 2.x Compatibility Note:**
> PyTorch (currently versions 2.0 through 2.8) has compatibility conflicts with NumPy 2.x on macOS, which can result in runtime initialization errors. To run this project successfully, you **must pin NumPy to version 1.x** (i.e. `numpy<2`).

We recommend using **`uv`** (a fast Python package manager) to run scripts without manual virtual environment configuration, or a standard virtualenv.

### Option A: Using `uv` (Recommended & Automatic)
The workspace has been fully initialized as a `uv` project. To run the tests using your configured environment, simply run:
```bash
uv run python test_project.py
```

If you ever need to re-initialize the environment or add new dependencies:
```bash
# 1. Initialize the project with Python 3.12 (required for PyTorch compatibility on Intel macOS)
uv init --bare --python 3.12 .

# 2. Add all required packages (specifying torch<=2.2.2 for Intel macOS compatibility)
uv add "torch<=2.2.2" "numpy<2" scipy tqdm jupyter nbconvert ipykernel matplotlib torchvision
```

### Option B: Using a standard Python Virtual Environment (`venv`)
Alternatively, create and activate a virtual environment manually:
```bash
# Create a virtual environment
python3 -m venv .venv

# Activate it
source .venv/bin/activate

# Install required dependencies (ensuring numpy is pinned to < 2)
pip install "numpy<2" torch torchvision scipy tqdm ipykernel matplotlib
```

## 3. Running the Automated Test Suite

We created an automated unit test suite `test_project.py` using Python's `unittest` framework. It verifies:
1.  **Normalization (`norm.py`)**: Checks boundary mappings and exact reconstruction.
2.  **SDE equations (`sde.py`)**: Confirms SDE noise scaling parameters compute correctly.
3.  **ScoreNet Model (`unet.py`)**: Tests neural network instantiation and shapes during forward passes.
4.  **Loss Function (`loss.py`)**: Validates loss calculation and backpropagation gradients.
5.  **Monte Carlo (`mc.py`)**: Tests actions, drift calculations, gauge cooling, and HMC transitions.
6.  **Observables (`observable.py`)**: Verifies jackknife error analysis, autocorrelation, and ESS.
7.  **Samplers (`sampler.py`)**: Verifies topological charge logic and field calculations.

### Run Tests:
Ensure your virtual environment is active, and run:
```bash
python test_project.py
```
Or run using `uv`:
```bash
uv run --python 3.12 --with torch --with 'numpy<2' --with scipy --with tqdm python test_project.py
```

Expected output:
```text
Ran 7 tests in 0.152s

OK
```

## 4. Running the Main Simulations

The project contains two Jupyter notebooks demonstrating the physics simulations:

1.  **`DM_SQ_U(1).ipynb`**:
    *   Generates configuration training data using HMC.
    *   Normalizes and prepares dataset.
    *   Trains the time-dependent U-Net score-based model (`ScoreNet`).
    *   Generates new samples using `MALA_sampler` guided by the trained model.
    *   Compares observables (plaquettes and topological charge) between HMC datasets and generated samples.
2.  **`langevin.ipynb`**:
    *   Runs standard Langevin dynamics on the CPU using Numpy.
    *   Compares purely mathematical Langevin sampling efficiency with custom parameters.

To run these notebooks:
1.  Launch Jupyter Lab or Notebook:
    ```bash
    jupyter notebook
    ```
2.  Open the desired notebook (e.g. `DM_SQ_U(1).ipynb`), configure your kernel (pointing to the environment where PyTorch and pinned NumPy 1.x are installed), and run all cells sequentially.
